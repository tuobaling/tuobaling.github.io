OpenClaw 使用 Codex Cli OAuth 回應Http 500

08 Apr 2026 • 3 min read

OpenClaw 使用 Codex Cli OAuth

Codex CLI 整合 GitHub Actions 錯誤日誌

問題概述

錯誤訊息：

PLAIN
代理程式嘗試連線至 OpenAI API（wss://api.openai.com/v1/responses）時收到 HTTP 500 錯誤

發生時間： 2026-04-08

影響範圍： GitHub Actions 上執行的 Codex CLI 工作流程

問題發生過程

第一階段：初始規劃

一開始的目標是讓專案能夠真正使用 Codex CLI。規劃的架構是：

PLAIN
Telegram → GitHub Issue → GitHub Claw Worker → Codex CLI (GitHub Actions)

第二階段：本機測試成功

在本機（Windows）測試 Codex CLI：

BASH
codex exec "say hi" -a never

結果：成功 ✅

PLAIN
model: gpt-5.4
provider: openai
approval: never

本機測試確認：

  1. Codex CLI 已正確安裝

  2. ChatGPT Plus 訂閱可正常運作

  3. 模型 gpt-5.4 可用

第三階段：部署到 GitHub Actions

嘗試 1：workflow 無 auth step

最初的 workflow 只有：

YAML
- name: Install OpenAI Codex CLI
  run: npm install -g @openai/codex@${{ vars.CODEX_CLI_VERSION || '0.118.0' }}

- name: 執行任務
  shell: bash
  env:
    CODEX_HOME: ".codex"
  run: |
    codex exec --json --yolo "..."

結果：HTTP 500 錯誤 ❌

原因：GitHub Actions runner 上沒有認證資訊。

嘗試 2：加入 auth step，但用錯路徑

YAML
- name: Restore Codex Auth
  shell: bash
  run: |
    mkdir -p ~/.codex
    echo "${{ secrets.CODEX_AUTH_JSON }}" | base64 -d > ~/.codex/auth.json
    chmod 600 ~/.codex/auth.json

同時 workflow 的執行任務 step 設定了：

YAML
env:
  CODEX_HOME: ".codex"

結果：HTTP 500 錯誤 ❌

原因：路徑不一致！

  • Codex 讀取：$CODEX_HOME/auth.json = ./.codex/auth.json

  • auth.json 寫入：~/.codex/auth.json（home directory）

嘗試 3：用錯變數命名空間

YAML
- name: Restore Codex Auth
  if: ${{ vars.CODEX_AUTH_JSON != '' }}  # ❌ 錯誤
  shell: bash
  run: |
    echo "${{ vars.CODEX_AUTH_JSON }}" | base64 -d > ~/.codex/auth.json

結果：Secret 讀不到，auth step 無效 ❌

原因：GitHub Actions 中 vars.XXX 是 Variables，secrets.XXX 才是 Secrets。

嘗試 4：GitHub Secret base64 損壞

手動貼上 base64 到 GitHub Secret 時，可能因為：

  • 字串太長被截斷

  • 特殊字元被錯誤處理

  • 貼上時漏掉結尾

結果：JSON 解析錯誤 **expected \\**,**** or \\**}**** at line 1 column 803**** ❌**

嘗試 5：最終成功 ✅

修正 workflow 檔案：

YAML
- name: Restore Codex Auth
  shell: bash
  env:
    CODEX_HOME: ".codex"
  run: |
    mkdir -p "$CODEX_HOME"
    echo "${{ secrets.CODEX_AUTH_JSON }}" | base64 -d > "$CODEX_HOME/auth.json"
    chmod 600 "$CODEX_HOME/auth.json"

並重新產生完整的 base64 字串，重新設定 GitHub Secret。

結果：成功 ✅

錯誤分析

錯誤 1：路徑不一致（核心問題）

原因： workflow 中設定了 CODEX_HOME: ".codex"，這告訴 Codex 去 ./.codex/ 找設定檔。但 restore auth step 把 auth.json 寫到了 ~/.codex/（home directory）。

修正： 使用相同的 CODEX_HOME 環境變數：

YAML
env:
  CODEX_HOME: ".codex"
run: |
  mkdir -p "$CODEX_HOME"
  echo "${{ secrets.CODEX_AUTH_JSON }}" | base64 -d > "$CODEX_HOME/auth.json"

錯誤 2：變數命名空間混淆

原因： GitHub Actions 有兩種變數儲存，Secrets 存在獨立的加密儲存區。

修正： 使用 ${{ secrets.CODEX_AUTH_JSON }}。

錯誤 3：base64 在 GitHub Secret 中損壞

原因：

  • auth.json base64 字串很長（約 8KB）

  • 在 GitHub 網頁介面貼上時可能被截斷

  • 或特殊字元處理出錯

修正：

  1. 本機重新產生 base64：powershell -Command "[Convert]::ToBase64String([IO.File]::ReadAllBytes('C:\\Users\\wunhu\\.codex\\auth.json'))"

  2. 確認 base64 可以正常解碼

  3. 在 GitHub Secret 介面完整貼上

正確的 workflow 設定

Step 1: 安裝 Codex CLI

YAML
- name: Install OpenAI Codex CLI
  run: npm install -g @openai/codex@${{ vars.CODEX_CLI_VERSION || '0.118.0' }}

Step 2: 還原 ChatGPT 登入權杖（關鍵！）

YAML
# 還原 ChatGPT 登入權杖（用於 GitHub-hosted runner）
- name: Restore Codex Auth
  shell: bash
  env:
    CODEX_HOME: ".codex"  # 必須與執行任務 step 相同
  run: |
    mkdir -p "$CODEX_HOME"
    echo "${{ secrets.CODEX_AUTH_JSON }}" | base64 -d > "$CODEX_HOME/auth.json"
    chmod 600 "$CODEX_HOME/auth.json"

Step 3: 執行任務

YAML
- name: 執行任務
  id: codex_task
  shell: bash
  env:
    CODEX_HOME: ".codex"  # 與 restore step 保持一致
  run: |
    codex exec --json --yolo "請將最終結果寫入 artifacts/${COMMENT_ID}/result.md"

GitHub Secret 設定方式

1. 取得 auth.json 的 base64 編碼

Windows：

POWERSHELL
powershell -Command "[Convert]::ToBase64String([IO.File]::ReadAllBytes('C:\\Users\\wunhu\\.codex\\auth.json'))"

macOS/Linux：

BASH
base64 -i ~/.codex/auth.json

2. 建立 GitHub Secret

  1. 到 Repo → Settings → Secrets and variables → Actions

  2. 點 New repository secret

  3. 名稱：CODEX_AUTH_JSON

  4. 值：貼上完整的 base64 字串（一字不漏）

  5. 儲存

3. 驗證 Secret 已正確設定

Secret 設定成功後，workflow 執行時可以看到：

PLAIN
Restoring Codex Auth...

且 auth.json 會出現在 $CODEX_HOME/ 目錄。

為什麼本機成功但 GitHub Actions 失敗？

本機環境

  • codex login 會把 auth.json 存到 ~/.codex/auth.json

  • Codex 預設讀取 ~/.codex/auth.json

  • 瀏覽器 OAuth 流程正常完成
