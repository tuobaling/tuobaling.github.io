Hugo + Notion + Github Action 踩坑記錄 v1

26 Mar 2026 • 2 min read

Hugo + Notion + Github Action Log

專案：

https://github.com/tuobaling/tuobaling.github.io/

踩坑：

  1. hugo 編譯完的public 要跟source code不同repository，否則部署完之後只會看到ReadMe.md

  2. 建立hugo.yaml 自動部署檔的時候，要改用gh-pages、github person access token再加上github 帳號才能做到跨repo push

  3. 一定要在本機先裝好hugo並打包好，才能在本機建置確認站台內容是否有問題

建立步驟：

本倉庫以 Hugo + Minimal Black 主題建置 {username}.github.io 靜態網站。

如需英文版導覽，請參考 README.md 。

資料夾架構

PLAIN
.
├── archetypes/                       - Hugo 內容模板（參考 [Commands](README.md#commands)）
├── assets/                           - 主題的 CSS/JS 與 Tailwind 輸出
│   ├── css/
│   └── js/
├── content/                          - Hugo 會渲染的 Markdown 文章/頁面
├── data/                             - 提供 layouts 使用的資料
├── layouts/                          - 自訂 Hugo 佈局
├── scripts/                          - 工具，例如 `notion-sync.py`（連回 [Technologies](README.md#technologies)）
├── static/                           - 原封不動複製到 `public/` 的靜態資源
├── themes/
│   └── minimal-black/                - Minimal Black 主題（詳見 [Theme](README.md#theme)）
├── .github/
│   └── workflows/
│       └── hugo.yml                  - GitHub Actions 工作流程（參照 [Deployment](README.md#deployment)）
└── hugo.toml                          - Hugo 設定（baseURL、params、menu）

安裝步驟

  1. 本機安裝 Hugo Extended（Tailwind 資產需要 Extended 版本）。

  2. 註冊 GitHub 帳號（若尚未註冊）。

  3. 設定 Notion 綁定。

  4. 克隆此專案並設定 secrets、baseURL。

  5. 建立你的 {username}.github.io 倉庫（將 {username} 換成你的 GitHub ID）。

使用技術

  • Hugo Extended - 處理 Markdown、套用主題，設定寫在 hugo.toml。

  • Minimal Black 主題 - 提供樣式與變數（見 themes/minimal-black/assets/css/base.css），並整合 Tailwind（參照 Theme ）。

  • GitHub Actions - .github/workflows/hugo.yml 編譯 ./public、發布 Pages，並推送 artifact 到 {username}/{username}.github.io（見 Deployment ）。

  • GitHub Pages - Hugo 生成的靜態網站托管平台。

  • Python - scripts/notion-sync.py（在資料夾架構中提到）用來和 Notion 同步。

常用指令

  • hugo server -D - 啟動含草稿的開發版。

  • hugo --minify - 產出 ./public。

  • python scripts/notion-sync.py - 同步 Notion 內容（需更新憑證）。

  • git status / git add / git commit - 進行 Git 作業後推至 main。

部署

  1. 執行 hugo --minify --baseURL ...，工作流程會將 ./public 上傳為 artifact。

  2. actions/deploy-pages@v4 發布 artifact 至此 repo 的 GitHub Pages。

  3. peaceiris/actions-gh-pages@v3 使用 secrets.PAGES_DEPLOY_TOKEN 推送相同 artifact 到 {username}/{username}.github.io。

可透過推送 main 或在 Actions 頁手動啟動 workflow 來重建。

主題

網站使用 Minimal Black Hugo 主題；更多主題操作與色彩設定請見： https://gitlab.com/tuobaling/hugo-minimal-black。

English reminder: color variables └──color-bg, └──color-surface, └──color-text, └──color-text-muted, └──color-border, └──color-accent live inside themes/minimal-black/assets/css/base.css.
